# numpy

# ---- Optional: web/social scraping (PRIMARY provider) -------------------
# ScrapeGraphAI is the primary scrape/search provider for the social + web
# connectors (Firecrawl is the fallback). Optional: if absent, those connectors
# transparently fall back to Firecrawl, then to cached replay. Install it (in a
# venv — it pulls langchain + Playwright) to enable live web/social/X scraping
# without a Firecrawl plan:
#
#   python3 -m venv .venv && . .venv/bin/activate
#   pip install scrapegraphai langchain-google-genai
#   playwright install chromium
#
# Run scan.py / preflight.py with that interpreter. The search() path (X/social)
# uses an LLM; by default it runs on Gemini Flash (fast, concurrent, ~cents/scan)
# — set GEMINI_API_KEY (Google AI Studio). For a free local fallback set
# SGAI_MODEL=ollama/<model> (no key, install Ollama); or SGAI_MODEL=openai/<model>
# + OPENROUTER_API_KEY. The scrape() path (page -> markdown) needs no LLM. Tune
# via SGAI_MODEL / SGAI_SEARCH_RESULTS / SGAI_MAX_SEARCHES / SGAI_MAX_TOKENS /
# SGAI_MAX_BROWSERS / OLLAMA_BASE_URL / SCRAPE_PROVIDER_ORDER.
#
# scrapegraphai
# langchain-google-genai   # only when SGAI_MODEL is a Gemini model
#
# ---- Optional: AWS Bedrock LLM (no API key — instance-role auth) ---------
# Run the search LLM and/or the Explainer on Amazon Bedrock (preferred for an
# AWS-native or GovCloud/FedRAMP deploy). Set SGAI_MODEL=bedrock/<model-id> and
# AWS_REGION; boto3 (already used elsewhere) resolves creds from the standard
# chain. See README §14 "LLM via AWS Bedrock".
#
# langchain-aws            # backs the Bedrock provider for scrapegraphai search
# boto3                    # Bedrock converse API for the Explainer agent
